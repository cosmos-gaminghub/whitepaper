# <a id="top"></a>GAME Whitepaper 2.0

# <a id="abstract"></a>概述
GAME是一个去中心化自治游戏平台，可以让玩家、开发者、投资人等全球各类热爱游戏的人群直接参与游戏运营。GAME Hub将具有价值的游戏内物品等数据代币化并以去中心化的方式进行管理。代币化物品可以存放在玩家自己的钱包当中，可以随时随意进行交易。换句话说，GAME 利用区块链实现了无需信任及无需同意的物品交易。 GAME 还将为游戏开发商提供多种模块化和可定制机制，显著降低去中心化游戏开发的门槛。社区管理对于这些渐进且适应性强的机制发挥至关重要的作用。为了让 GAME 继续成为一个创新的去中心化游戏平台，根据需要依次升级几乎所有元素至关重要。去中心化自治平台 GAME 在社区成员评论现有参数，例如市场、质押奖励、交易平台上的激励、对现有功能和应用程序的修改以及实行新功能和应用程序时，需要社区成员参与到发言、讨论、投票等当中。
“我们将建立一个去中心化自治平台，并同时实现可持续增长和生态系统的扩展。”
GAME 项目本身是一款出色的 MMO（大型多人游戏）RPG。

# <a id="wmgu"></a>系统特点
## <a id="cdados"></a> 由社区主导的去中心化运作机制
与传统型的、从上到下存在指挥链并进行相应管理的普通中心化组织结构相比，GAME的每个独立社区成员都以自愿行动而进行积累并且自主发展为一个去中心化组织。 GAME作为一个去中心化自治组织，调整权益质押和流动性供给的激励函数、调整市场和交易的费用函数以及决定如何使用累积的社区资金，都会通过GAME Hub 的链上管理进行决定。由于没有任何的绝对管理者做出决定，因此可以防止形成一个独裁组织，并且可以构建多元化的生态系统。

## <a id="hgpbfts"></a> 玩家如何受益
在传统中心化结构的游戏当中，通过充值以及打游戏获得的物品所有权并不属于玩家，而是属于游戏运营公司。因此，玩家无法主动买卖游戏内物品，退坑游戏时，手中的物品一文不值。对于游戏公司来说，不保留物品价值会直接提升短期内的销售盈利，所以说这种机制是故意对玩家产生不利。为了即使在退坑游戏后也能保留玩家在游戏中努力获得的价值，这时需要有一种机制，让游戏中获得的物品的所有权归属于玩家，并且可以自由的进行交易。因此，GAME 提供了一种对玩家友好的机制，允许用户通过 GAME Hub 将游戏物品进行代币化，从而摆脱游戏公司的集中管理。这允许玩家自愿将物品转让给好友或通过 GAME Market Place 进行出售。以这种方式出售的物品将是 GAME Hub 的基轴货币 $GAME，它也可以通过 Cosmos 生态系统内的 DEX 转换为其他数字货币。
通过启用这一系列的自由经济活动，玩家在退坑GAME Ecosystem中的游戏A并开始新的游戏B时，依然可以保留游戏A的物品的价值。将日新月盛的游戏空间中的物品作为数字资产进行管理必然能够产生各类经济活动，玩家将能够体验并享受到更加自由和具有吸引力的新一代去中心化虚拟空间。不让在游戏中的努力徒劳无益——而通过游戏赚钱，对于热爱游戏的玩家来说，将会成为最大的乐趣。

## <a id="hgdedamtg"></a> 让游戏开发商可以轻松开发和运营的机制
GAME将通过四种方式支持游戏开发商进行游戏开发和运营，以积极的方式消除游戏开发和运营时面临的障碍。首先是游戏开发和运营时的资金筹集问题。小户游戏开发商，即便对开发游戏充满了创意，也往往面临资金问题的困扰。若有一种方式让这些资金不足的游戏开发商展示他们的创意并可以从支持他们的人手中筹集到资金，这个问题便得到解决。 GAME 将提供一个融资平台，可以轻松的为新游戏筹集开发资金。在这个平台上，玩家和投资者可以看到各类创意并做出投资决策。这种通过数字货币的支付方式毫无疑问的在全球范围内实现了快速融资并大大助理于游戏开发。
其次是支付系统和发布平台的提供。在游戏开发、发布并进到盈利阶段时，一定需要一个支付系统和发布平台。但是，没有独家平台的游戏开发商为此不得不依赖各大平台，并且需要支付大笔手续费。一般来说，开发商需要对平台支付 30% 的游戏销售额手续费，而在某些国家手续费会高达至 90%。而GAME 可以大幅度降低高昂的手续费。使用区块链技术的去中心化支付系统，无需中心化的第三方即可实现高安全性，且开放性的公链，不会在支付系统上手续任何费用。当然，为了维护链条，每个用户在发出交易时都必须支付交易手续费，但对于游戏开发商的平台支付系统手续费是完全免费。此外，在发行上，GAME 将大幅度降低的成本。具体的费用比例应由社区成员的管理来决定，但如果费用的设定对游戏开发商是友好的，那么几个百分点应该是比较可观的。以往来说这类方式所收取的费用，会直接计入到平台游戏的利润当中，但在GAME平台里，会返还给社区。这将带动整个 GAME 的发展，反过来又会间接增加自身游戏用户的数量，带动社区开发者产生并带来积极的影响。极低的费用可以说是一个去中心化自治组织的独家优势，把生态系统的扩张放在首位，不会仅仅寻求短期的利润。
第三是为去中心化游戏的开发提供各种功能和应用。如果游戏开发商需要从头创建区块链的话会使其门槛太高，不切实际。 因此，GAME 准备并提供所必备的功能和应用程序，让无法从头创建区块链的开发商实现去中心化游戏的开发。 此外，游戏开发商可以通过与GAME社区内工程师直接进行沟通，降低去中心化游戏开发的准入门槛。
最后是有关于免费提供例如角色和物品之类的开放资产。一般来说游戏中经常出现的“药水”和“草药”等回收物品，“Excalibur”和“Gungnil”等武器，以及人类、矮人、精灵和哥布林等模型角色，提前进行2D或3D设计后，以FT 和 NFT的形式发布于 GAME Ecosystem并可以在各类游戏中使用。通过允许 GAME Ecosytem 中的所有游戏使用这些资产，可以在游戏开发中针对特定代币持有者进行精确营销并节省成本。为了增强开放资产和开放资产设计提供激励，今后会在社区管理中进行更多的讨论，。

# <a id="faa"></a> 功能与应用
介绍未来将在 GAME 中实现的具体功能与应用程序。

## <a id="itemtoken"></a> Item Token
对游戏内物品进行代币化是玩家管理游戏内物品所有权并允许他们自由交易的最重要功能。游戏开发商可以使用 GAME Hub 生成两种主要类型的代币。 Fungible （可替代）代币和 Non-Fungible（不可替代）代币。前者是使用在游戏中类似药水等物品的代币格式，对应以太坊中的ERC-20代币。作为 Fungible 代币发行的游戏物品没有区别，可以一一替换。然而，许多游戏当中都会存在独特的游戏内物品，具有个性化价值，使游戏空间更加精彩。具有独特等级、攻击力、技能的游戏内物品将作为元数据保留其状态并成为不可回收的代币。这对应于以太坊中的 ERC-721 代币，使其成为世界上唯一不可替代的代币。这两种代币都可以让用户作为 GAME Hub 上的数字资产拥有，并且可以使用 GAME MarketPlace 和游戏内托管功能进行交易。

## <a id="dex"></a> DEX
当前的大多数数字货币交易都存在与有运营实体的交易所进行交易。这些交易所被称为集中式交易所 (CEX)，因为它们使用集中式的决策方法。在这样的CEX中，用户的资产首先从用户管理的钱包发送到交易所的钱包，余额在交易所的数据库中进行管理。此时，由于资产本身是由交易所存储的，用户需要信任交易所，而数字资产完全依赖于第三方（交易所），就会处于高风险状态。事实上，曾经在中心化交易所中，确实出现过交易所拿走用户资产或受到黑客攻击客户资产的情况。此外，由于CEX存在单点故障，存在访问集中和DDoS攻击问题往往导致的并非是区块链瘫痪而是交易所服务器瘫痪，从而无法进行交易。另一方面，由 GAME 实现的、没有运营实体、以去中心化方式执行交易的交易所称为去中心化交易所（DEX）。用户始终可以自己管理自己的资产，在交易时，用户从自己的钱包中将代币发送到交易地址并进行存款，这样就可以在无需信任第三方的情况下进行交易。
如上所述，GAME Hub 将构建一个 DEX，创造一个玩家可以自由买卖代币的环境，以实现代币（Fungible Tokens）的去中心化交易，无需通过第三方。 GAME Hub 的 DEX 允许使用 GAME Hub 平台上的任何代币进行交易，使用 $GAME 作为关键货币。代币和 $GAME 交易费率基于 AMM 算法，在供需匹配的点上进行平衡和决定。 供应越少，需求越高，价格就会越高。 然而，这些代币没有流动性就不能交易且缺乏实用性，因此创造大量具有流动性的激励设计至关重要。对于流动性供应商分配多少 $GAME 等决策会通过社区管理做出决定。

## <a id="marketplace"></a> MarketPlace
GAME MarketPlace允许购买、出售和拍卖 GAME 生态系统中分布的 NFT。 玩家可以把游戏中实用性高的代币根据需求进行交易。如果能把在游戏中努力获得的稀有物品以高价卖掉，以玩游戏的方式也是可以赚钱。，可以可为了保持游戏内物品的价值，市场的机制可以说是占据了举足轻重的地位。

## <a id="tradings"></a> 托管交易
托管交易功能，实现了在与游戏中遇到的其他玩家进行物品交易时的无障碍、无风险。代币发送到对方的托管专用地址，当双方代币存入完成后，自动执行模块中定义的代币转账传输函数，完成托管交易。需要额外说明的是，交换的物品不一定是一种物品对应另一种物品，也可以按照预先设定的种类和数量来制作托管列表。因此，可以执行例如A Item x 1 + B Item x 2 + C Item x 3和D Item x 1 + E Item x 1的托管交易。在链上将托管交易进行自动化后，可以让交易时无需信任屏幕另一边的陌生人，给游戏的自由度做出了巨大贡献。

## <a id="funding"></a> Funding
面向全球发行的游戏，众多是由各大游戏厂商转包的中小游戏公司开发，并由各大厂商发行。然而近年来，个人可以轻松构建应用程序，制作游戏必需隶属与大厂商这一事实开始有了变革。 GAME Hub 将建立一个系统，允许任何人，包括个人游戏开发商和资金匮乏的中小型游戏公司，为游戏项目提供资金。需要得到资助的游戏开发商可以向社区提交白皮书和提案，清楚地展示新游戏的概念和对投资者的利益。社区成员将根据提交的材料在社区内进行严格的审查和讨论，并根据自己的判断，决定是否投资该项目。投资者将 $GAME 发送到提供资金的地址并接收本地代币。原生代币持有者将有权利在游戏上线时优先获得稀有物品、自动获得游戏销售额的X%或获得$GAME等各类福利中受益。通过这种融资机制，投资者可以支持新游戏，开发商也可以筹集资金，实现顺利开发。这样的机制形成了一个又一个不断吸引新游戏加入 GAME 生态的循环，从而带动新用户的加入，并有助于 GAME 平台的成长。此外，GAME中的这种融资机制可以解决过去投资游戏时存在的问题。过去，游戏基金成立时，投资者在结算后只能获得其利润的仅仅几个百分点的分红。这时候，由于向投资者分红效果不佳，游戏公司有时会为了尽可能减少对投资者的分红而有意增加费用，故意给利润造成压力。由于此类欺诈，投资者的利润逐渐减少，导致投资者不愿投资游戏，阻碍了整个行业的发展。再者，对于一个没有完善的分红制度的游戏进行投资的情况下，投资者在几个月内收到分红的情况并不少见，这对游戏行业的投资来说是一个桎梏。但是，在 GAME 中，通过使用 Funding 功能，可以始终实现游戏中产生的销售额的分红机制。通过使用这种去信任化的奖励分红系统，开发者方面无需担心欺诈，投资者可以比以往更加积极地将游戏项目作为投资目标。

## <a id="claim"></a> Claim
Claim 模块是管理一个模块账户中被管理的代币按照持有比例分配给 GAME Hub 上发行的 Claim 代币持有者的机制的模块。
例如，当游戏合约的部分销售额转入模块账户时，Funding 功能允许 Claim 代币的持有者根据其持有比例进行索要额度。

## <a id="gameid"></a> GAME ID
GAME 会增加一个功能，可以为每个地址设置一个 GAME ID。让用户直接使用熟悉的 ID 来完成汇款，而无需在发送代币时输入复杂的地址。此外，使用这种方式设置的GAME ID，只需一个签名即可注册和登录游戏。 这将降低不熟悉数字货币的普通游戏玩家的门槛，并为用户提供比以往更轻松的游戏体验。

## <a id="random"></a> 随机数生成模块
在 GAME 中，我们为了防止运营商对游戏中的扭蛋机制进行欺诈性行为，导致扭蛋机制中的爆率出现作假，在此起到关键的是链上随机数的生成。众所周知，在计算机上生成完全随机数是一项非常困难的任务，其中一种方法是使用 TEE（可信执行环境）通过秘密计算生成随机数。 TEE 是计算机内部的一个特殊区域，其作用类似于黑匣子，可秘密执行计算。 TEE 用于日常生活中像安装在智能手机中的指纹识别器之类的各类硬件上。正是 Secret Network 中的计算模块，让 TEE 机制实现了 Cosmos 生态系统中的链上机密计算。网络上的秘密节点可以利用这个安全区域来保护保密协议中使用的数据。 GAME 旨在利用这种机制创建一个随机数生成模块。同时其他具有创新的隐私解决方案包括多方计算、同态加密和机密公共集计算等等，但这些并不适合在当前应用中实际使用。可扩展性、性能和安全性始终处于一种权衡关系之中，我们需要继续探索和研究各种解决方案。

## <a id="explorer"></a> 管理器
管理器可以查看每个地址的内容，包括查看交易。在使用基于 PoS 的共识算法的 GAME Hub 中，有一个名为 Validator 的节点可以生成区块。 此外，GAME Hub 具有Onchain Governance 作为通过投票进行社区决策的功能，并且可以执行 IBC 交易。 管理器需要支持这些功能，但实际上 GAME 中已有的管理器支持governance模块和 IBC 模块，以便可以查看投票结果和 IBC 交易内容。

## <a id="platform"></a> 平台
构建一个方便用户，能够在游戏生态系统中将游戏做出排列的一个平台。 此时，在游戏平台上市需要在GAME中进行管理审批。并且可以向社区池提出建议，例如支付 $GAME 或共享原生代币，以通过投票获得认可。

# <a id="structure"></a> 结构
Game Hub 使用 Tendermint、Cosmos SDK 和 Cosm Wasm 构建特定于游戏的区块链。

## <a id="aotendermint"></a> Tendermint的优势
通过使用 Tendermint 将其实现一条独自链，一般认为可以享受以下优点。由于 Tendermint 可以根据 ABCI 的规格标准自由实现接口，因此它可以在区块链上实现自己的state-machine（状态机）。此外，应用程序可以自由选择验证器的数量、Gas和代币模型。这对于社区主导的区块链项目至关重要。 Tendermint 的共识算法是广义上的 PoS，因此可以实现绝对的 TPS。此外，与以太坊不同的是，无需与其他项目竞争资源，这是主要优势之一。由于各种利益相关者的参与，以太坊中的错误无法轻易修复，但在基于tendermint的专有链中，错误可以通过社区内的管理以共识的方式处理。此外，即使游戏物品的规格发生变化，也可以通过 CosmWasm 的迁移功能轻松升级，此项将在后面介绍。

## <a id="aocosmossdk"></a> Cosmos SDK 的优势
可以使用 Cosmos SDK 创建自定义模块。利用GAME Hub 可以创建出自定义的 NFT 模块。 此外，Cosmos SDK 中的 IBC 模块和链下流程中继器可以轻松与外部基于 Tendermint 链进行交互。 这允许用户通过在 Cosmos 生态系统内的多个 DEX 上列出买卖 $GAME 和其他代币，包括在 Cosmos Hub 上实施的 DEX。

## <a id="aocosmwasm"></a> CosmWasm 的优势
CosmWasm 是一个可以插入 Cosmos SDK 的模块。 CosmWasm 是一个开源项目，作为智能合约的多链解决方案从头被设计和构建，并已在各种项目中广泛使用。 从游戏开发商的角度来看，从零开始构建一条独特链并将其融入游戏是存在很大的难度的。 因此，GAME Hub 准备了一个环境，可以使用 CosmWasm 执行专门用于游戏的通用智能合约。 游戏开发商可以使用 GAME Hub 上的 NFT 模块，在 CosmWasm 的合约端自由编写游戏代码。

### <a id="amaic"></a> 关于 CosmWasm 中使用的 Actor 模型
Actor 模型是构建去中心化系统时的设计模式之一，每个Actor都被赋予了管理自己内部状态的权限，是一种不能直接重写其他状态的机制。如果需要重写另一个状态，会被要求通过发送消息来改变它。在 CosmWasm 中，通过任何合约直接调用另一个合约，可以防止例如在以太坊的智能合约中发生的 DAO 事件一样的智能合约规范中造成漏洞的攻击（重入攻击）。对恶意代码的内联函数调用会产生安全漏洞。 CosmWasm 采用的 Actor 模型方法允许任何合约重返同一交易中执行的请求消息列表。也就是说，一个合约可以请求后期传输（例如，托管释放）或调用另一个合约。若一个未来的消息显示失败，那整个交易包括合约的更新状态将被恢复原状。

## <a id="cois"></a> Interoperable Sovereignty的特点
为了暂时解决以太坊[附录A]的问题，出现了许多以牺牲去中心化为代价而专注于性能的平台[附录B]。还有一些具有新安全模式的平台，例如 Polkadot [附录 C]。 但当面临于黑客等安全威胁时，由于平台的依赖性，导致的了其严重的损坏。在这种情况下，一个重要的概念就是“互用主权（interoperable sovereignty）”。
sovereignty，狭义上是指应用程序是独立性，不受其他资源的影响。 如果sovereignty没有实现，比如币安智能链实现的合约存在漏洞，那么引用该合约的应用程序可能会被同样的漏洞攻击。再者，如果一个项目爆红，那其他项目也会面临gas飙升、出现交易处理速度缓慢等问题。 就以这些应用程序的安全性和可扩展性而言，sovereignty是一个重要因素。
一般认为，在未来10年里将会有各类区块链项目诞生，真正形成“区块链互联网（Internet Of Blockchains）”。链和代币将被创新、消失、增长、收缩、分裂和整合。在这种动态当中，每条链和代币都形成了自己的社区。这种与各个社区中产生的互动也会成为一个重要的因素。通过IBC，与具有不同角色的社区之间形成的交流将会变成不具有单点故障的松散耦合式的连接。专为使用 Cosmos SDK 的应用程序区块链不必为此担忧，因为这是一个独立的区块链，并且可以通过 IBC 进行发送和接收代币等通讯。

# <a id="conclusion"></a> Conclusion
区块链作为无需通过中心化第三方的P2P（Peer to Peer）批准交易的技术，以围绕比特币而崛起，具有防篡改性和高度实用性，实现了作为降低运营成本的新技术。不仅在普通汇款和支付上，同时也挑战于国际汇款和物联网当中，目前实证研究正在进行在各个领域当中。然而，无论任何项目都至今仅都限于试验阶段，仍然处在一个发展中的阶段内。 2017年，各类区块链项目开始启动，活跃了ICO的资金调动。然而，也有一些诈骗项目，即便无需使用区块链，也通过ICO强行与区块链关联起来筹集了大量的资金。尽管区块链的本质价值是“无需信任”和“无需许可”，但一个忽视其本质的项目只能大同小异于传统项目，并不能带来任何新一代的改革。毫不夸张地说，在真正理解了区块链的意义之后，支持项目的社区的存在、代币的实用价值、产品本身就是掌握区块链项目命运的关键所在。社区的本质是开源、非营利和一个自下而上的生态系统，与现有的自上而下的商业生态系统截然不同。代币代表价值的所有权，虽然只是电子数据，但在创建虚拟世界的新经济圈中发挥着重要作用。产品则是使用区块链技术的成果本身。如果没有社区或代币价值，即使只做出了产品，也不能充分发挥区块链项目的魅力。在区块链用例稀缺的情况下，真正能够决定区块链项目未来的是应当是一个用例明确的项目。 GAME Hub作为一个区块链项目，是基于实用价值的区块链，在全球范围内尝试投入区块链技术实际应用的同时，建立其作为游戏用例的地位，并作为一个扩展应用发展。为此，GAME Hub 将专注于实现游戏内物品代币，为用户建立物品所有权，以及在 GAME Hub 中实现 DEX 以促进用户之间的交易，并在 GAME Hub 中为游戏开发商开发 Funding 系统和 SDK。GAME Hub 是基于社区的运营，将 $GAME 作为 GAME Hub 内的基准货币，为代币增值，打造去中心化的游戏平台，将改变传统游戏的玩法，实现虚拟世界中的代币经济；另一方面，GAME Hub 也起到了向社会展示一种崭新的游戏作用方式。无论玩什么游戏，很快就会感到乏味，这可能是因为忽略了游戏的前景；毕竟，游戏可能会被认为只是娱乐的一部分，只不过是一个与现实世界无关的空洞事物。然而，随着电子竞技的发展，传统游戏应有的方式正在世界各地被重新审视。同时，通过使用区块链技术，虽然在还处于开发阶段，但是一定程度上已经在防篡改性和高度可用性的同时实现了以去中心化的方式存储数字数据。游戏打得再好也一无是处的时代已经结束，玩游戏给现实世界带来兴奋，获得别人羡慕和优越感才是游戏行业中新时代的标签。在这样的背景之下，GAME Hub作为一个去中心化的游戏平台，从玩家的角度提供全新的游戏价值，虽然每个玩家对游戏的印象、鉴赏、优越感等因人而异，但我们的目标是让已经变得模棱两可的游戏重新构建为一个真正可以被通晓的“压倒性自由与现实”的虚拟空间。相信新时代一定是由我们新一代年轻人来开创。

## <a id="appendix-a"></a> 以太坊的挑战
由于以太坊网络是一个可以实现所有去中心化应用的平台，所有项目都同时在以太坊上进行开发。因此，以太坊网络持续承受着需要处理大量交易的压力。 2021年11月，约17万笔未确认交易发生不断，交易费用也在不断上涨。其背景被认为是 DeFi 等新应用的出现和 NFT 泡沫的复燃为原因。
TPS（Transaction per Second），相比比特币确实有所改善，但以太坊每秒15笔交易的数量显然对于未来预计会增加的网络来说是不够的，并且在需要立刻进行结算的游戏行业当中，会导致用户体验的显著下降。对于使用链下技术的改进计划依然在研讨阶段，其实施的时间和可行性尚不清晰。另外需要注意的是，系统回滚是为了应对 The DAO 问题，硬分叉会导致社区部分分裂。由于各种项目的介入，以太坊的使用会涉及一些问题和风险，而为了实现Game Hub所追求的免费虚拟空间，会专门为游戏打造具有高用户体验的新框架。

## <a id="appendix-b"></a> 模仿以太坊的新平台
Binance Smart Chain（BSC）由全球领先的交易所Binance发起，是一个分叉以太坊客户端Go-ethereum的项目，共识部分从传统的PoW转为DPoS，共有21个validator。 BSC实际上是一条私有链，通过与以太坊相比大幅度降低了共识成本，成功处理了大量交易，成为薄饼（pancakeswap）等各种项目使用的主要平台。
同样，有Polygon、Solana等多条实用型的链可以处理大量交易，可以说目前平台处于一个动荡状态。

## <a id="appendix-c"></a> Cosmos 和 Polkadot 的区别
虽然 Cosmos 和 Polkadot 都有一些共同点，可以轻松创建互用操作的区块链，但在安全模型方面存在显著差异。在 Polkadot 中，有一个叫做中继链（relaychain）的父链和一个叫做平行链（parachain）的子链。中继链管理每个连接的平行链的全局状态。在中继链里，区块的最终确定性是通过名为 GRANDPA 的共识算法而获得。 Polkadot 采用了一种称为“共享安全（shared security）”的模式，平行链侧可以享受中继链的安全性。另一方面，在 Cosmos Network 中，每条链都是独立的，自己链的安全性由为每条链设置的验证器来进行保障。此外，在 Polkadot 的情况下，目前可以连接到中继链的平行链数量为100，每个平行链都可以通过称为平行链的拍卖机制连接到中继链。这时需要大量购买和锁定DOT。使用 Cosmos Network，任何人都可以自由构建自己的链并连接到其他链当中。
